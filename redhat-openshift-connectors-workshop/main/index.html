Red Hat OpenShift Connectors Workshop

Introduction

Welcome to the Red Hat OpenShift Connectors workshop.

As a developer of applications and services, you can use Red Hat OpenShift Connectors to create and configure connections between OpenShift Streams for Apache Kafka and third-party systems.

OpenShift Streams for Apache Kafka is a cloud service that simplifies the process of running Apache Kafka. Apache Kafka is an open-source, distributed, publish-subscribe messaging system for creating fault-tolerant, real-time data feeds. OpenShift Streams for Apache Kafka enables you to add Kafka data-streaming functionality in your applications without having to install, configure, run, and maintain your own Kafka clusters.

You can use OpenShift Connectors to configure communication between Streams for Apache Kafka instances and external services and applications. OpenShift Connectors allow you to configure how data moves from one endpoint to another without writing code.

The following diagram illustrates how data flows from a data source through a data source connector to a Kafka topic. And how data flows from a Kafka topic to a data sink through a data sink connector.

Figure 1. OpenShift Connectors data flow

At the moment there are more than 50 source and sink connectors available. These include source and sink connectors to a variety of cloud services based on the awesome Camel-K technology, as well as source connectors for databases based on the popular Debezium project for Change Data Capture. The complete list of available connectors to date can be found here.

In this workshop you will:

  • Provision a Kafka instance using OpenShift Streams for Apache Kafka.

  • Configure the Streams for Apache Kafka instance for use with Red Hat OpenShift Connectors.

  • Create a Connectors instance as a data source.

  • Create a Connectors instance as a data sink.

  • Create a Connectors instance to capture data change events from a cloud database.
